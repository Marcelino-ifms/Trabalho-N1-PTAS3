Sparks Fly - Gerência de Usuários

Usamos nessa API as seguintes tecnologias:

  • Sequelize
  • Sequelize cli
  • Express JS
  • Postgres SQL
  • Thunder Client
  • Nodemon

Funcionalidades:

Cadastro de usuários

O cadastro de novos usuários no banco de dados através do verbo "create" vindo do sequelize que foi instalado e usado ao longo do desenvolvimento. Isso ocorre de modo que ao acessar uma rota "post", informando os dados do usuário é feito seu cadastro no banco, como informado no arquivo "routes", com isso, a função "create users" é executada, e os dados serão salvos no banco de dados com um id identificador.

Consultar usuários

A consulta dos usuários é efetuada após a requisição através do método get, como escrito no arquivo routes. Ao acessar essa rota, a partir do sequelize é possível fazer a consulta usando o verbo "findAll" para a listagem de todos os usuários cadastrados, executada na função "searchUsers".

Alterar usuários

A função de alterar usuários, também é feita através de uma requisição, porém, na rota "put" datada no arquivo routes, utilizando o verbo "Update" do sequelize. Deve-se portanto informar o id do usuário à ser alterado na url da api, após, informa-se os dados alterados que serão atualizados no banco de dados, através da função "updateUser".

Deletar usuários

Para deletar um usuário, a requisição é feita pela rota "delete", que é executada no verbo "destroy" do sequelize na função "deleteUser", usando como parâmetro o id do usuário cadastrado.

Autenticar usuários

Por fim, estabelecemos também a funcionalidade de autenticação de usuários, que é feita com os dados de email e senha, utilizando o verbo "findOne" do sequelize na função "authenticatedUser", que verifica a partir do parâmetro "user", se o email e password fornecidos pelo formulário batem com os cadastrados no bando de dados, se sim, o usuário ganhará um token e será autenticado. Essa função é executada em uma rota "post".
